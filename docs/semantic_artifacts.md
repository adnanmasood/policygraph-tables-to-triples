# Semantic Artifacts

`ontology/policygraph.ttl` is the OWL/RDFS ontology graph. It defines classes,
properties, ranges, labels, comments, and SKOS concept schemes.

`mappings/policygraph.rml.ttl` is the RML mapping from CSV rows to RDF subjects,
predicates, objects, datatypes, and IRIs.

`rdf/policygraph-data.nt` and `rdf/policygraph-data.ttl` are generated data graph
serializations. They contain explicit facts materialized from the CSVs.

`rdf/policygraph-combined.ttl` is generated by merging the ontology graph and data
graph. SPARQL application queries use this graph.

`shapes/policygraph.shacl.ttl` is the SHACL shapes graph. It defines closed-world
quality constraints for required values, datatypes, references, and cross-record
business checks.

`shapes/validation-report.ttl` and `shapes/validation-report.txt` are generated
validation reports from pySHACL.

`queries/*.rq` are reusable SPARQL query files. SELECT queries produce result
sets. The account context CONSTRUCT query produces a bounded RDF subgraph.

The ontology graph describes meaning. The data graph describes source facts. The
combined graph supports query-time context. The shapes graph validates data
quality. Query result sets are tabular answers, while constructed subgraphs are
RDF snippets suitable for downstream semantic workflows.
